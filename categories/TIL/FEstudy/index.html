1. Function Advanced

2. Closure

  • 외부함수의 실행이 끝나서 외부함수가 소멸된 이후에도 내부 함수가 외부 함수의 변수에 접근할 수 있는데, 이러한 메커니즘을 클로저라 한다.
  • 클로저 자신에 대한 접근, 외부 함수의 변수에 대한 접근, 전역 변수에 대한 접근.

2-1. Private variable

  • 클로저는 객체의 메소드에서도 사용할 수 있다.
  • 객체의 외부에서는 접근 할 수 없는 내부에 감춰진 속성이나 메소드를 의미
  • 프로그래밍에서 어떠한 데이터를 아무나 수정하는 것을 방지 하기 위해 사용된다. 안전한 변수 사용.

3. HOF(Higher-Order Function)

  • 함수형 프로그래밍 : 순수 함수와 보조 함수의 조합을 통해 로직 내에 존재하는 조건문과 반복문을 제거하여 복잡성을 해결하고 변수의 사용을 억제하여 상태 변경을 피하려는 프로그래밍 패러다임
  • 하나 이상의 함수를 인자로 받는다.
  • 함수를 결과로 반환한다.
  • 함수를 다루는 함수

참조

  • https://poiemaweb.com/js-array-higher-order-function
  • https://dev-momo.tistory.com/entry/HigherOrder-Function-%EC%9D%B4%EB%9E%80-%EB%AC%B4%EC%97%87%EC%9D%B8%EA%B0%80

4. Callback

  • 나중에 호출되는 함수
  • 응답을 받아야 실행되므로 순서대로 실행하기 위해 사용된다.
  • 어떤 이벤트가 발생했거나 특정 시점에 도달했을 때 시스템에서 호출하는 함수
  • 콜백함수는 클로저다.

1. This

  • 컨텍스트(실행 환경) 생성 시 정의된다.
  • 스크립트가 로드 돼서 코드가 실행되면 모든 것을 포함하는 전역 컨텍스트가 생긴다. 전역 컨텍스트 외에도 함수 호출 시마다 함수 컨텍스트가 생긴다.
  • This는 해당 함수 호출 방식에 따라 바인딩 되는 객체가 달라진다.
  1. 함수 호출
  2. 메서드 호출
  3. 생성자함수 호출
  4. apply/call/bind 호출
  5. 이벤트

1-1. 함수 호출

  • 전역컨텍스트에서나 함수기본 호출에서의 this에 바인딩되는것은 window 전역객체이다.
  • 일반함수, 메서드, 콜백함수 어디서 선언되었든 해당 내부 함수의 this는 모두 전역객체이다.
  • strict mode를 사용해서 함수를 정의하는 경우에는 다음과 같이 this는 undefined 값이 된다.

1-2. 메서드 호출

해당 메서드를 호출한 객체가 바인딩된다.

1-3. 생성자함수 호출

생성자함수를 선언하면 빈 객체를 생성하고 함수 내 this는 이 빈 객체를 가리킨다.
생성자를 통해 생성된 인스턴스 자신이 바인딩된다.

1-4. apply/call/bind 호출

첫번째 매개변수로 넘긴 값이 this에 바인딩된다.

1-5. 이벤트

이벤트의 콜백함수의 this에 이벤트를 건 객체가 바인딩된다.

2. OOP

  • 프로그램 내에서 표현하고자 하는 일들을 객체로 모델링하는 것을 말한다.
  • 상태나 행동을 프로그래머의 의도에 맞게 추상화하여 데이터를 만드는 프로그래밍 패러다임.
  • 객체 인스턴스는 class를 통해 만든다.
  • 객체지향 프로그래밍은 보다 유연하고 유지보수성이 높은 프로그래밍을 하기 위해 필요하다.

생성자 함수

  • 클래스의 자바스크립트 버전, 단순히 프로퍼티와 메소드를 정의
  • new 키워드와 호출하면서 새로운 객체 인스턴스 생성

참고

  • https://developer.mozilla.org/ko/docs/Learn/JavaScript/Objects/Object-oriented_JS
  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Introduction_to_Object-Oriented_JavaScript

3. Class

  • ES2015에 추가된 개념으로 새로운 객체지향 상송 모델을 제공하는 것이 아니라 객체를 상속하고 다루는 데 있어 훨씬 더 단순하고 명확한 문법을 제공

3-1. Class 정의

  • Class는 함수이다. class문법도 clas표현식, class 선언 방법을 제공한다.
  • class선언은 호이스팅이 일어나지 않는다.

3-2. Class body & Class method

  • Class body는 중괄호로 {}로 묶여 있는 안쪽 부분이다. strict mode로 실행된다.
  • constructor는 생성된 인스턴스를 초기화 하기 위한 메소드. 클래스안에 한 개만 존재 가능
  • Class method는 메소드 정의를 위한 더 짧은 구문(method definitions)이 도입되었고, 그 방식대로 body안에 메소드명 그대로 정의한다.

참고

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Classes

4. Call, Apply, Bind

4-1. Call

  • 다른 객체의 메소드를 빌려와 호출하는 방식. 주어진 this값과 arguments로 함수를 호출한다.
  • 메소드가 non-strict mode 코드의 함수일 경우, null 과 undefined 가 전역 객체로 대체되며, 기본 값은 제한된다.
  • arguments는 인수 리스트로 받는다.

4-1. Apply

  • call 메소드와 동일하나, arguments는 인수들의 단일 배열로 받는다.

4-3. Bind

  • 다른 객체의 메소드를 빌려와 주어진 this값과 arguments로 새로운 함수를 반환한다.

1. Object

1-1. 객체의 기본정의

  • 모든 사물 하나하나
  • 키와 값의 쌍
  • 프로퍼티의 순서 없는 컬렉션
  • 사물의 상태나 행동에 대해 정의한 데이터

속성(Property)

  • 키:값의 관계를 속성이라 한다.

키(Key)와 값(Value)

  • 속성명을 키라 하고 문자열만 가능하다.
  • 값은 어떤 값이나 가능하다. 속성값이 함수인 것을 메소드라 한다.

1-2. 객체 생성

기본객체의 생성자 함수 이용

리터럴방식

생성자함수 이용

  • 함수는 정의 될 때 함수에 생성자 자격이 부여되고, 해당 함수에 Prototype Object가 생성 및 연결된다.
  • 생성자 함수 : 정의한 함수에 new 키워드를 붙여 변수에 호출하는 것. 생성자 함수는 일반 함수와의 구분을 위해 첫글자는 무조건 대문자로 작성한다.
  • “현재 생성된 객체”를 인스턴스라고 하는데, 인스턴스의 프로퍼티를 재설정하여도 원본 객체에는 영향이 가지 않는다.
  • 생성자함수는 동일한 프로퍼티를 갖는 인스턴스를 만드는 일반적인 방법으로, ‘객체를 초기화(공통 상태의 처음으로 돌아가는 거) 한다’고 말할 수 있다.

2. Standard built-in Objects

  • 자바스크립트가 기본적으로 가지고 있는 객체, 전역 범위의 여러 객체

3. Prototype

  • 어떠한 객체가 만들어지기 위해 그 객체의 모태가 되는 객체를 프로토타입이라 한다.
  • 모든 함수는 prototype 프로퍼티를 가진다.
  • 같은 생성자로 만들어진 인스턴스들은 이 원형 객체를 공유한다.
  • 동일하게 사용되는 프로퍼티는 불필요한 메모리 낭비를 줄이기 위해 prototype으로 만든다.

1. JavaScript Variable Memory

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Memory_Management

메모리 생존주기는 프로그래밍 언어와 관계없이 비슷하다.

  1. 필요할때 할당한다.
  2. 사용한다. (읽기, 쓰기)
  3. 필요없어지면 해제한다.

1-1. 자바스크립트 메모리 생성주기

  1. 값을 선언할 때 메모리를 할당한다. 몇 가지 함수 내에서도 호출을 통해 메모리 할당이 일어나며, 몇 가지 메서드도 새로운 값이나 오브젝트를 담기 위해 메모리 할당이 일어난다.
    *메서드 : 객체의 값이 함수 일 때 이를 메서드라고 한다.
  2. 변수나 오브젝트 속성 값을 잃고 쓸때 값 사용이 일어난다. 또, 함수 호출시 함수에 인수를 넘길 때도 일어난다.
  3. 자바스크립트는 자동으로 가비지 콜렉션을 수행하므로 더 이상 사용하지 않을 값을 찾아내서 해당 값이 차지하고 있던 메모리를 회수한다.

1-2. 가비지 콜렉션

가비지 콜렉터는 어떤 값이 더이상 사용되지 않는지, 사용될 가능성이 있는 값이 무엇인지 추적하여 메모리 회수 대상을 정한다.
가비지 콜렉터는 항상 필요없어진 메모리만을 해제하지만 모든 필요없어진 메모리를 해제하는건 아니다.

주로 쓰이는 가비지 콜렉션 알고리즘

  • 표시하고 지우기
  • 참조 카운팅

2. Hoisting

  • https://developer.mozilla.org/ko/docs/Glossary/Hoisting

var 키워드가 사용된 선언문이나 함수 선언문(function(){})이 scope의 첫줄로 끌어올려지는 현상을 일컫는다.

let과 const가 hoisting이 발생하지 않는 건 아니다. 두 키워드는 블록 스코프 단위이다.
변수가 만들어 지고 TDZ가 생성되고, 변수가 실제 있는 위치에 도달 할 때까지 엑세스 할 수 없다.
*TDZ : 일시적 사각지대(Temporal Dead Zone; TDZ)

  • https://poiemaweb.com/es6-block-scope

3. Function

  • https://poiemaweb.com/js-function

함수란 어떤 작업을 수행하기 위해 필요한 문들의 집합을 정의한 코드 블록이다.
자바스크립트의 함수는 일급 객체이다.

일급 객체

  1. 변수나 데이터 구조 안에 담을 수 있다.
  2. 파라미터로 전달 할 수 있다.
  3. 반환값으로 사용 할 수 있다.
  4. 런타임에 생성될 수 있다.

3-1. 반환값

  • return 문으로 값을 반환 할 수 있다.
  • return문이 없을 경우에는 undefined를 반환한다.
  • 모든 함수는 값을 반환하며, 함수 자체가 값이 된다.

3-2. 매개변수

  • Argument : 전달해주는 값
  • Parameter : 전달 된 값은 받아들이는 변수

3-3. 순수함수

어떤 외부 상태도 변경하지 않고, 부수효과가 없는 함수를 칭한다.
*부수효과 : side effect, 외부의 상태를 변경하는 것 또는 함수로 들어오는 인자를 직접 변경하는 것

2. Value, Expression, Statement

2-1. 값(Value)

만들고 변경할 수 있는 데이터, 형(Type)을 표현하는 수단
모든 값은 타입을 가지며 타입은 값의 종류를 포함하는 더 큰 개념으로 확장된다.

값의 종류

  • 숫자 - Number
  • 문자열 - String
  • 논리값 - Boolean
  • 값이 정의되지 않은 - undefined
  • 값이 없는 - Null
  • 숫자가 아닌 - NaN
  • 심볼 - Symbol
  • 객체 - Object

2-2. 식(Expression)

하나의 값으로 표현된다.

  • https://www.bsidesoft.com/?p=760

*리터럴 : 더 이상 나눌 수 없는 값.

2-3. 문(Statement)

의미상 한줄 한줄 각각의 명령, 지시문. 문은 파싱된 이후에 아무것도 남지 않게 되므로 값으로 쓸 수 없다. (Return을 할수 있는건 값, 식)
ex. 조건문, 반복문, 제어문, 식 문, 변수선언문 …

3. Type (Primitive, Reference)

3-1. 기본값(Primitive)

참조되지 않고 복제되며 immutable한 자료형

  • string
  • number
  • boolean
  • undefined
  • null
  • NaN

3-2. 참조값(Reference)

복제되지 않고, 원본은 하나만 유지된 상태로, 주소만 공유되는 것(함수, 객체, 배열)
초기값을 설정할 때 해당 값의 메모리를 할당 받고, 값을 사용 할 때나 함수 호출 시 할당된 메모리를 읽고 쓰기만 한다.

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Data_structures
  • https://weicomes.tistory.com/133
  • https://poiemaweb.com/js-immutability

4. Truthy & Falsy

boolean 타입이 와야 하는 자리에 다른 타입의 값이 와도 실행되는 데 true로 되는 어떤 값들을 truthy, false로 취급 되는 어떤 값들을 falsy라고 부른다.
false, null, undefined, 0, NaN, ‘’(빈문자열)을 제외한 모든 값들은 truthy이다.

  • https://developer.mozilla.org/ko/docs/Glossary/Truthy
