1. This

  • 컨텍스트(실행 환경) 생성 시 정의된다.
  • 스크립트가 로드 돼서 코드가 실행되면 모든 것을 포함하는 전역 컨텍스트가 생긴다. 전역 컨텍스트 외에도 함수 호출 시마다 함수 컨텍스트가 생긴다.
  • This는 해당 함수 호출 방식에 따라 바인딩 되는 객체가 달라진다.
  1. 함수 호출
  2. 메서드 호출
  3. 생성자함수 호출
  4. apply/call/bind 호출
  5. 이벤트

1-1. 함수 호출

  • 전역컨텍스트에서나 함수기본 호출에서의 this에 바인딩되는것은 window 전역객체이다.
  • 일반함수, 메서드, 콜백함수 어디서 선언되었든 해당 내부 함수의 this는 모두 전역객체이다.
  • strict mode를 사용해서 함수를 정의하는 경우에는 다음과 같이 this는 undefined 값이 된다.

1-2. 메서드 호출

해당 메서드를 호출한 객체가 바인딩된다.

1-3. 생성자함수 호출

생성자함수를 선언하면 빈 객체를 생성하고 함수 내 this는 이 빈 객체를 가리킨다.
생성자를 통해 생성된 인스턴스 자신이 바인딩된다.

1-4. apply/call/bind 호출

첫번째 매개변수로 넘긴 값이 this에 바인딩된다.

1-5. 이벤트

이벤트의 콜백함수의 this에 이벤트를 건 객체가 바인딩된다.

2. OOP

  • 프로그램 내에서 표현하고자 하는 일들을 객체로 모델링하는 것을 말한다.
  • 상태나 행동을 프로그래머의 의도에 맞게 추상화하여 데이터를 만드는 프로그래밍 패러다임.
  • 객체 인스턴스는 class를 통해 만든다.
  • 객체지향 프로그래밍은 보다 유연하고 유지보수성이 높은 프로그래밍을 하기 위해 필요하다.

생성자 함수

  • 클래스의 자바스크립트 버전, 단순히 프로퍼티와 메소드를 정의
  • new 키워드와 호출하면서 새로운 객체 인스턴스 생성

참고

  • https://developer.mozilla.org/ko/docs/Learn/JavaScript/Objects/Object-oriented_JS
  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Introduction_to_Object-Oriented_JavaScript

3. Class

  • ES2015에 추가된 개념으로 새로운 객체지향 상송 모델을 제공하는 것이 아니라 객체를 상속하고 다루는 데 있어 훨씬 더 단순하고 명확한 문법을 제공

3-1. Class 정의

  • Class는 함수이다. class문법도 clas표현식, class 선언 방법을 제공한다.
  • class선언은 호이스팅이 일어나지 않는다.

3-2. Class body & Class method

  • Class body는 중괄호로 {}로 묶여 있는 안쪽 부분이다. strict mode로 실행된다.
  • constructor는 생성된 인스턴스를 초기화 하기 위한 메소드. 클래스안에 한 개만 존재 가능
  • Class method는 메소드 정의를 위한 더 짧은 구문(method definitions)이 도입되었고, 그 방식대로 body안에 메소드명 그대로 정의한다.

참고

  • https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Classes

4. Call, Apply, Bind

4-1. Call

  • 다른 객체의 메소드를 빌려와 호출하는 방식. 주어진 this값과 arguments로 함수를 호출한다.
  • 메소드가 non-strict mode 코드의 함수일 경우, null 과 undefined 가 전역 객체로 대체되며, 기본 값은 제한된다.
  • arguments는 인수 리스트로 받는다.

4-1. Apply

  • call 메소드와 동일하나, arguments는 인수들의 단일 배열로 받는다.

4-3. Bind

  • 다른 객체의 메소드를 빌려와 주어진 this값과 arguments로 새로운 함수를 반환한다.
